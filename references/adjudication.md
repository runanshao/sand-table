# 反事实推演裁定规则

当用户的选择 ≠ 史实，剧情进入"我编"的领域。这里是 AI 最容易乱来的地方——可以让你的烂决策侥幸成功、也可以让你的好决策惨死，**全凭一念**。若放任，整个技能的诚实性崩塌。故强制以下裁定纪律。

## 铁规
1. **打标**：每段反事实输出开头标注【反事实推演 · 非史实 · 置信度 X/5】。
2. **不溯及计分**：反事实段落只是后续剧情，**不改变已经在决策时刻完成的评分**（评分早于揭示、早于反事实）。
3. **按因果 + 基率裁定，不按戏剧性裁定**：
   - 后果由"当时真实存在的力量对比、概率、各方真实动机"推导，不由"什么更精彩/更解气"推导。
   - 用历史基率锚定：同类局面在历史上多半怎么收场？给出后果时附一句依据。
4. **不奖励侥幸、不惩罚正确**：好决策即使在反事实里也该多数情况导向好结果（除非小概率风险命中——那要明说"这是那 X% 的尾部事件"）。烂决策不该被我编得侥幸通关。
5. **承认不确定**：分叉越远，置信度越低。两步以上明确提示"越往后我越是在猜，可信度下降"，并适时把用户拉回下一个**史实锚点**重新对账。
6. **可被质疑**：用户质疑某个反事实后果不合理时，重新用因果+基率检验，认错优先。

## 何时致死的判定（生存层）
当用户选择导向危险后果，按因果+基率估"致死概率"，不按戏剧性：
- 列出此决策触发的真实威胁（埋伏/断粮/兵变/政变/被俘…）。
- 给一个区间估计："在多少成的合理推演里，这步会让你死/被俘/全盘崩。"附一句依据（同类局面的历史基率）。
- **不奖励侥幸**：致死概率高 + 用户这局没死 = 明说"运气吊着你"，不抬高判断分。
- **不惩罚正确**：致死概率低 + 用户这局死了 = 明说"史实骰子背，判断没错"。
- 记入日志的是「本该死锚点」（第一个致死概率过高的决策点），不是「实际死锚点」。

## 置信度标尺
- 5/5：紧贴史实已知的力量对比，几乎必然
- 3/5：方向可信，细节是推测
- 1/5：远离史实分叉、纯推演，仅供过瘾

## 实操
反事实推一般不超过 1–2 步，然后引导：
> "再往后就纯属我编了。我们跳回下一个**有史实可对账**的决策点——那里你的判断才能再被真正检验。"
