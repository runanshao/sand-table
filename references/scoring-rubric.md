# 五维评分细则

**最高原则：评的是"决策时刻的推理质量"，不是结果，也不是是否复刻史实。**
评分必须在揭示结局**之前**完成，且每条评语引用信息边界卡里的具体项。

每维 0–2 分（精推模式 10 分制；快推只用单维直觉 0–2）。

## 1. 信息利用（用足已知了吗）
- 0：忽略了摆在面前的关键线索 / 凭空臆想
- 1：用了部分信息，漏了重要的
- 2：把信息边界卡里的已知项基本榨干，推理扣着证据走

## 2. 风险定价（最坏情况标对价了吗）—— 判断vs运气的核心维度
- 0：无视下行 / 把小概率灾难当不存在（项羽式）；或被恐惧放大风险到瘫痪
- 1：意识到风险但定价粗糙
- 2：明确估了最坏情况的概率与代价，并据此调整下注大小
> 这一维直接支撑"判断vs运气"判定：一个决策即便这次输了，只要它在当时是正期望，就是好决策。

## 3. 反证强度（自我反驳过吗）
- 0：一冲动 / 只找支持自己的理由（确认偏差）
- 1：想到了反方但没认真对待
- 2：决策前主动构造了最强反方论证，并回应了它

## 4. 目标对齐（服务真实目标，还是被情绪带跑）
- 0：被面子、愤怒、贪婪、沉没成本带跑，决策偏离真实目标
- 1：大方向对，但夹带了私心/情绪
- 2：清楚自己真正要什么，决策严格服务它（哪怕难看、憋屈）

## 5. 盲点意识（知道自己不知道什么吗）
- 0：把"我不知道"当"不存在"；或没识别出哪个才是真转折点（被诱饵带走）
- 1：意识到有盲区但没行动
- 2：明确点出关键未知、并为之留了余地 / 主动去补；正确识别了真转折点

---

## 揭示后：判断 vs 运气归因（必做）
揭示结局后，对**你的决策**和**史实人物的决策**各做一句：
- 「判断好 + 结果好」= 实至名归
- 「判断好 + 结果衰」= 虽败犹荣（决策没错，输在运气/信息不可得）
- 「判断烂 + 结果好」= 侥幸（最危险——别被它强化坏习惯）
- 「判断烂 + 结果烂」= 活该
并显式给一句：**"本次结果里，运气大约占 X 成。"**

> 训练目标不是追求"结果好"，是把自己往"判断好"那一列搬。结果交给概率。

## 生存层：致死概率 + 死亡对账

满足"何时死/撑多久"的需求，但**不破坏"判断≠运气"脊椎**——靠的是把"过程致死"和"单局实际生死"分开。

### 每锚点：致死概率估计
揭示后给一句：**"在多少成的合理推演里，你这个选择会让你死/被俘/全盘崩。"**
- 这是**过程锚定**估计（基于当时力量对比与基率），不是单局一次性结果。
- 高致死概率 + 你活下来了 = **运气吊着你**，不是判断好。
- 低致死概率 + 你死了 = 史实骰子背，判断没错。
- 维护生存状态：健康 / 危殆 / 出局（累计致死风险触发）。

### 终局：本该死 vs 实际死（核心对账）
摆两条线：
- **你实际撑到第几锚**（含运气）
- **你的决策本该让你死在第几锚**（纯判断：第一个致死概率过高的决策点）
两者一对：
- 实际 >> 本该 → 你被运气吊着，别把"还活着"当判断好。
- 实际 << 本该 → 你判断没错，输在史实运气，别改习惯。
- 实际 ≈ 本该 → 判断与命运对齐，最实在。

> 记进 decisions.jsonl 的是「本该死锚点」，不是「实际死锚点」——因为我们练的是判断，不是赌运气。
