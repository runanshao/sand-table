# 场景构造格式（现场拆历史 / 建库都用这个）

一个场景 = 一条**锚点链**。每个锚点是一个决策时刻。

> 结构化场景分 `scenario-blind/v1|v2` + `scenario-sealed/v1|v2` 双文件（schema 见下）。
> **v2 = v1 + 资源盘（blind）+ 执行摩擦（sealed）**，供「谋主沙盘模式」用；v1 场景照常可跑经典流程。

## 锚点结构
```
## 锚点 N：<不剧透的中性标题>
- 时间地点
- 【你知道的】（当事人此刻确实掌握的信息）
- 【你不知道的】（明确告诉用户"你不知道X"——制造真实的信息盲区）
- 【你以为是真、但可能错的】（当事人的误判/情报偏差）
- 决策问题：<只问"你怎么办"，不给暗示哪个对的选项措辞>
- ——以下揭示前不可见——
- 史实选择 + 出处（《...》）+ 置信度
- 实际结果 + 运气成分估计
- **致死条件 / 生存影响**：哪些选择会触发死/被俘/崩盘，及其致死概率区间 + 依据
- 对照体推理（揭示前据信息边界卡定稿）
- **可迁移原则**：这一锚点压成的一条跨领域判断原则（供现实决策桥使用）
- **参谋团**（供参谋团模式）：本锚点真实在场/适配的 2–3 位历史人物 + 其**有记载的立场**（无记载则标"据其思想体系推测"）。至少一人持"反对玩家倾向"的最强反方。
- 演义/通俗版差异（如有）→ 标红 + 做成判断课
```

> 史源纪律（裁判诚实）：每个史实主张须可附出处与置信度；拿不准就标"史料有争议/记不准"，绝不编出处。

## v2 资源盘（blind 顶层 `assets:` · 谋主沙盘模式的地基）
"真实世界里所有资源和人都有属性，不会丝滑执行你的意图，甚至反噬"——把这件事**预先结构化**，
而不是让 AI 现场即兴（现场即兴 = 顶级编造）。

```yaml
assets:
  天时: ['季节/气候/时机——按当事人当时的认知']
  地利: ['地形/水道/距离/补给线']
  兵粮: ['兵力构成、粮草还能撑多久——按当时认知，可以含当事人的误判']
  利益网: ['哪些集团/家族/派系的利益绑在哪一头']
  掣肘: ['后方/朝廷/舆论/身份给行动套的枷锁']
  人和:                      # 可用之人，逐人带属性
    - 名: 某人
      角色: 谋士/部将/封疆…
      能力: 一句话，按当时名声与履历（非上帝视角）
      你以为的忠诚: 当事人此刻的判断（允许是误判）
      私利: 他自己的算盘绑在哪
      掣肘: 用他时受什么限制
      反噬信号: 当事人此刻可感知的危险苗头（绝不剧透他后来真做了什么）
```

**资源盘铁规**：
1. 每一条都限定**当事人当时知道或以为的**——上帝视角信息一律不进 assets。
2. 每一条在 sealed 对应锚点能找到史源支撑；编不出史源的条目不进盘。
3. 不许出现任何结局性词汇（泄题闸 `scenario_lint.py` 会扫 assets 文本）。
4. `反噬信号` 写的是**苗头**（如"其家族在邺城有把柄在审配手里"），不是**结局**（如"他后来叛逃"）。

## v2 执行摩擦（sealed 每锚点可选 `execution_friction:`）
```yaml
execution_friction: |-
  这一步史实里"人和资源实际怎么走样"：谁打了折扣、谁阳奉阴违、谁反噬、
  哪条粮道/地理约束实际咬人了。附出处与置信度。
```
揭示时与用户的盘点、谋主的六查做**三线对账**：你漏了什么、谋主漏了什么、史实多狠。

## 构造铁规

### 防泄题（盲点⑥之一）
- 锚点标题、信息边界卡的**颗粒度必须一致**。不要在"关键锚点"突然给更多细节——那等于告诉用户"这里有坑"。
- 真实的当事人不知道哪一刻是转折点。不要用编排暗示重要性。

### 掺诱饵（盲点⑥之二）
每条锚点链里混入：
- **看着重要、其实不重要**的决策点（消耗用户注意力，训练取舍）。
- **看着平淡、其实是转折**的决策点。
- 让"**识别哪个才是真正的转折点**"本身成为一道判断题（计入"盲点意识"维度）。

### 对抗选题幸存者偏差
《明朝那些事》《三国演义》只讲"事后看很关键"的桥段。直接照搬会让用户养成"每刻皆史诗"的错觉。构造时**主动补入真实存在过的平庸/诱饵决策**，还原"在日常噪音里识别信号"的真难度。

### 史实纪律
- 锚定正史；演义只作选题来源与氛围。
- 拿不准的细节（具体兵力、谁说了哪句话）标置信度或"史料有争议"，**不编**。

## 现场拆一段历史的步骤
1. 定主角、定时间窗、定 3–5 个决策时刻。
2. 对每个时刻倒推"此人当时的信息边界"——剔除一切他不可能知道的。
3. 至少塞 1 个诱饵锚点。
4. 备好每个锚点的史实结局、出处、运气估计、对照体推理。
