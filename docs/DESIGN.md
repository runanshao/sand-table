# 史鉴推演 — 设计文档（v1）

> 生成：2026-06-01 ｜ 状态：待用户验收样板后规模化

## 一句话定位
一个**判断力训练**工具：把你代入某段历史的关键决策点，扮演当事人，在"只知道他当时知道的"前提下做决策、写理由，然后拿你的判断跟**史实**对账，区分"判断对"和"运气好"，跨场景累积出你自己的判断模式。

**沉浸是手段，对账是目的。** 历史的唯一不可替代价值：结局已知 = 自带标准答案 = 真实的反馈回路。

## 与已有技能的边界
- `hu-deduction`（胡式推演）：现实决策，用"胡"的视角。
- `史鉴推演`：历史情境，信息隔离 + 史实对账 + 判断力档案。
不重叠。

## 核心铁规：对账只认史实
- 评分一律锚定**正史/史料**：曹操→《三国志》；明朝→《明史》及一手史料。
- **演绎版（《明朝那些事儿》《三国演义》）只做两件事**：① 选题菜单（挑哪些精彩决策点）② 氛围调味。
- 凡**演绎 ≠ 史实**处强制标红，且把这个差异本身做成一堂判断课（"你的决策被通俗叙事滤镜带偏了没？"）。

## 架构：三角色严格分离（防后见之明泄露）
1. **情境官**：只描述当事人此刻能感知的信息；禁止提及未来、禁止事后聪明的措辞。
2. **史官/裁判**：仅在用户提交决策后出场，掌握史实与评分。
3. **对照体（史官AI）**：基于同样受限信息独立给出一个高质量决策样本，作对照基准。
- 每锚点开头出**信息边界卡**：【你知道的】【你不知道的】【你以为是真但可能错的】。

### 裁判诚实度护栏（盲点①：裁判=AI，无人监督）
本技能全部角色由同一 AI 扮演，AI 有两大固有失真：讲错史实、把后见之明泄露进对照体。强制护栏：
- **史实可溯源**：关键史实主张标注出处（如《三国志·武帝纪》《明史·于谦传》）+ 置信度；不确定处明说"不确定/史料有争议"，**绝不编造确定性**。
- **对照体反泄题**：对照体的决策**只能依据信息边界卡**生成，且必须在揭示史实**之前**定稿，并写出其推理链供用户检验（防止"我知道结局→让对照体显得更聪明"）。
- **欢迎质疑**：每锚点提示用户可质疑 AI 的史实陈述；被质疑时优先核对、认错，而非辩护。

## 一次推演流程（7步）
0. 选场景 + 选扮演谁 + 锁起始锚点 + **选档位（快推/精推）**
1. 角色简报：你是谁、处境、目标、约束、手里的牌。**只给处境，不给人格设定**（盲点④：防止你去"演人设"而非用自己的判断；明确指令"用你自己的判断，别演谁"）
2. 信息边界卡
3. 你做决策 + 写理由（精推=强制可证伪理由；快推=一句话）
4. 揭示前先打分（过程评分，不被后果污染）
5. 揭示三线：史实人物怎么选 / 实际结果 / 史官AI怎么选 + **估"本次结果里运气占几成"**（盲点③）
6. 四线对账（你 / 史官 / 史实人物 / 实际结果）→ 当场区分判断 vs 运气
7. 进入下个锚点。**计分锚定"决策时刻的推理质量"，与你是否复刻史实无关**（盲点②）。选择=史实→沿真实历史推进；≠史实→切【反事实推演】标签（明示置信度），剧情继续但该锚点的决策**照常计分**
末. **每锚点评分即时追加写入档案**（盲点⑦：防中途退出丢数据），战役结束做整局小结

### 双档位（盲点⑤：摩擦过大会掐死循环频率）
- **快推**：一句话理由 + 轻评（单维直觉打分）。保日常高频。
- **精推**：强制可证伪理由 + 五维评分 + 运气归因。保硬仗深度。
默认快推，用户可随时升级到精推。

## 评分五维（过程导向，不评结果好坏）
| 维度 | 问的是 |
|---|---|
| 信息利用 | 摆在面前的线索用足了吗 |
| 风险定价 | 最坏情况标对价了吗 |
| 反证强度 | 决策前自我反驳过吗 |
| 目标对齐 | 服务真实目标还是被情绪/面子带跑 |
| 盲点意识 | 知道自己不知道什么吗（含：能否识别"哪个才是真转折点"）|

**判断 vs 运气的判定方法（盲点③，不是口号）**：评判基准是"**在当时信息与概率分布下，该决策的期望值**"，**不是**"这次成没成"。每次揭示后显式估"本次结果里运气占几成"，并据此把"结果好但判断烂/判断好但结果衰"两类如实标出。结果好坏单独标注，**绝不并入判断分**。

## 判断力档案（兑现"训练"的部分）
每次评分+盲点**即时**追加写入 `profile.md`；跨场景找出你反复出现的判断模式（盲点/偏好/强项）。
落地"判断力 = 决策→复盘 的循环频率 × 诚实度"。
**样本量诚实（盲点⑧）**：模式归纳必须标注样本数与置信度；n 过小（如<5个同类锚点）只列"观察"、不下"规律"，明说"还看不出稳定模式"——绝不拿噪音当规律。

## 场景构造铁规（盲点⑥：锚点选取本身会泄题 + 选题幸存者偏差）
- **掺诱饵锚点**：每个场景混入"看着重要其实不重要"与"看着平淡其实是转折"的决策点，信息颗粒度保持一致——不在关键点突然给更多细节。让"识别哪个才是真转折点"本身成为判断题。
- **对抗选题偏差**：《明朝那些事》《三国演义》只讲"事后看很关键"的点，照搬会养出"每刻皆史诗"的错觉。构造时必须主动补入真实存在过的平庸/诱饵决策。

## v1 范围（YAGNI）
- 只做 **2 个样板场景**，验证味道：
  - **曹操 / 官渡之战**（选官渡而非赤壁：官渡在《三国志》中记载扎实、演义污染少，更适合做"对账史实"的首样板；赤壁的借东风/连环计多为演义，留作后续并作"演义vs史实"教学样本）。
  - **明朝 / 于谦·北京保卫战**（南迁 vs 死守、立新君、调兵——《明史》记载清楚，决策点密集）。
- 散场模式（AI 现场报一段）：v1 用同一套 scenario-format 即兴生成，不预置。
- 暂不做：多人对战、可视化、自动评分以外的统计图表。
