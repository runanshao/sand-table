# 史鉴推演 · 谋主沙盘模式 — 设计文档（v3）

> 生成：2026-06-10 ｜ 状态：实施中
> 在 v2 闭环内核（blind/sealed 物理墙 + 写序铁规 + 五维 + 闭环）之上加一层"资源与人的摩擦力"，不动地基。

## 用户原始诉求
假如我是曹操/袁绍/刘备，在每个史书场景里盘点手里有什么资源、有什么可用之人、有什么掣肘；真实世界中所有资源和人都有属性，不会丝滑无保留地执行意图，甚至反噬；我输入自以为完善的方案，AI 计算我盘点资源（天文/地理/人心/利益/兵粮/可用之人）的疏漏、对手的谋划、盟友背叛的条件。AI = 谋主。

## 对诉求的三处修正（不修正则项目自毁）

| 原诉求 | 陷阱 | 修正 |
|---|---|---|
| AI 瞬间算出一切疏漏 | 判断外包：用户判断力不增长，依赖增长 | **盘点先行**：用户先盘（用谁/靠什么/防什么反噬）并定稿，谋主才出对账单；「用户漏的 − 用户盘到的」差集是训练信号，喂进现有五维（信息利用/盲点意识/风险定价） |
| AI 集中世界顶级智慧 | AI 同样集中顶级编造能力（细节级史实张口就来） | 资源/人物属性/反噬信号**预先结构化进 blind 文件**（限定"当事人当时可感知"），谋主压力测试**只许引用资源盘+信息边界卡条目**，散场局的临时推断必须标"推测" |
| 揭示前替用户算对手谋划/背叛条件 | 泄题：谋主若暗示史实结局（"小心叛逃者"），对账作废 | 谋主只做**结构推演**（从盲区数据推可能性，不指向史实）；史实中真实发生的执行走样/背叛封在 sealed 新字段 `execution_friction`，评分落盘后由 historian 揭示，做**三线对账**（你的盘点 vs 谋主的盘点 vs 史实） |

## 新增物：四件

### 1. schema 升级（向后兼容，v1 场景照常可跑）
**`scenario-blind/v2`** = v1 + 顶层 `assets:`（资源盘，情境官可见，供盘点环节）：
```yaml
assets:
  天时: ['当事人当时对季节/气候/时机的认知…']
  地利: ['地形/水道/距离/补给线…']
  兵粮: ['兵力构成、粮还能撑多久（按当时认知，可含误判）…']
  利益网: ['哪些集团/家族的利益绑在哪头…']
  掣肘: ['后方/朝廷/舆论/身份给行动套的枷锁…']
  人和:                    # 可用之人，逐人带属性——"人不会丝滑执行意图"的载体
    - 名: 某人
      角色: 谋士/部将/封疆…
      能力: 一句话（按当时名声/履历，非上帝视角）
      你以为的忠诚: 当事人此刻的判断（可以是误判）
      私利: 他自己的算盘绑在哪
      掣肘: 用他时受什么限制
      反噬信号: 当事人此刻可感知的危险苗头（不是剧透他后来真叛了）
```
铁规：assets 全部限定**当事人当时知道或以为的**；每条可在 sealed 对应锚点找到史源；不许写任何结局性信息（泄题闸覆盖）。

**`scenario-sealed/v2`** = v1 + 每锚点可选 `execution_friction:`：
```yaml
execution_friction: |-
  这一步史实里"人和资源怎么走样"：谁打了折扣、谁阳奉阴违、谁反噬、
  哪条粮道/地理约束实际咬人了。附出处与置信度。
```

### 2. SKILL.md 新增「谋主沙盘模式」
流程（嵌进现有 0–8 步，写序墙原样）：
1. 信息边界卡之后、决策之前：摆出资源盘 → **用户先盘点**：方案 + 用谁执行 + 靠什么资源 + 防什么反噬 + 对手会怎么反制 + 盟友什么条件下会背叛。盘点定稿。
2. **谋主六查**（AI 出对账单，只许引用资源盘+信息边界卡）：天时、地理、人心、利益、兵粮、用人；外加两问：对手视角反推（站到对面席位用对面的资源盘想一轮）、盟友背叛条件（利益网里哪根线断了他就走）。每条疏漏标注引自资源盘哪一项。
3. 用户修订或坚持 → 可证伪理由 → 五维评分落盘（盘点疏漏计入信息利用/盲点意识/风险定价的评语依据）→ historian 揭示（含 execution_friction）→ 三线对账。
4. 谋主诚实铁规：不引用资源盘外的"史实"；结构推演与史料严格分层；不暗示史实走向。

### 3. 席位（seat）
同一战役可换边扮演，资源盘各席独立。首发三席凑齐用户点名人物：
- **曹操席·官渡**（升 v2）：兵少粮缺但谋士给力——练"劣势下用人敢不敢赌"。
- **袁绍席·官渡**（新建）：全部优势资源 + 每张牌都有毒（沮授田丰之谏与囚、郭图审配派系、许攸的私利与家族把柄、张郃高览的处境）——"人有属性、会反噬"的最佳教材。
- **刘备席·夷陵**（升 v2）：被排除在外的人（诸葛亮、赵云的反对）、带在身边的人（黄权的分营之谏）、三峡地理与情绪掣肘。

### 4. lint / historian 升级
- `scenario_lint.py`：接受 v1/v2；v2 校验 assets 六类齐全、人和逐人字段齐全；sealed v2 的 execution_friction 出现时该锚点 citations 仍强制；泄题 token 闸照常扫 assets 文本。
- `agents/historian.md`：揭示清单加 `execution_friction`；与 `~/.claude/agents/historian.md` 逐字同步。

## 不做（YAGNI）
- 不给人物属性数值化打分（忠诚度 70/100）——虚假精度，违反降虚假精度铁规。
- 不加第六个评分维度——DIMS 单一来源在 stats.py，加维度断历史曲线；盘点疏漏走现有三维的评语。
- 不做独立新 skill——会复制评分/闭环/隔离地基，双源漂移。
- 不动 stats.py / render.py / 校准金标集。

## 验收
1. `python scenario_lint.py` 全 PASS（含新建袁绍席、升级后的曹操席/刘备席）。
2. v1 旧场景（古巴单文件、未升级场景）不受影响。
3. historian 两处副本 diff 为空。
